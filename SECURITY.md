# Security Policy

## Supported versions

Only the latest code on `main` (and the latest GitHub Release, when releases exist) is supported with security fixes.

## Reporting a vulnerability

PM2 Matrix can start, stop, and restart PM2 processes and stream their logs. Treat authentication bypass, session flaws, and remote code execution–class bugs as critical.

**Do not open a public GitHub issue for security vulnerabilities.**

Report privately using [GitHub private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability) / Security Advisories for this repository:

https://github.com/manfredjekblad/pm2-matrix/security/advisories/new

Include enough detail to reproduce the issue (affected version or commit, steps, impact).

## Response

Reports are reviewed in good faith. Acknowledgement and status updates are sent when possible; there is no fixed SLA. Please allow reasonable time before any public disclosure.
